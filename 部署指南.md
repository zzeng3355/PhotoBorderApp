# Photo Border App 部署指南

## 方案一：本地编译（推荐，最快速）

### 前置条件

| 软件 | 版本要求 | 下载地址 |
|------|---------|---------|
| Android Studio | 最新版 | https://developer.android.com/studio |
| JDK | 17+ | Android Studio自带 |
| 手机或模拟器 | Android 8.0+ | — |

### 步骤

#### 1. 安装 Android Studio

1. 访问 https://developer.android.com/studio
2. 下载对应系统的安装包（Windows/Mac/Linux）
3. 运行安装程序，一路下一步即可
4. 首次启动时选择 **Standard** 安装（自动下载SDK）

#### 2. 导入项目

1. 解压 `PhotoBorderApp_v1.1.zip`
2. 打开 Android Studio
3. 点击 **Open** → 选择解压后的 `PhotoBorderApp` 文件夹
4. 等待 Gradle 同步（首次需要下载依赖，约5-10分钟，需要网络）

> 如果同步失败，检查网络连接，或尝试点击 **File → Sync Project with Gradle Files**

#### 3. 连接手机

**方式A：真机调试（推荐）**

1. 手机开启 **开发者选项**：
   - 设置 → 关于手机 → 连续点击"版本号"7次
2. 开启 **USB调试**：
   - 设置 → 系统 → 开发者选项 → USB调试（打开）
3. 用USB线连接电脑和手机
4. 手机上弹窗选择 **允许USB调试**

**方式B：模拟器**

1. Android Studio 点击 **Device Manager**（右侧边栏）
2. 点击 **Create Device**
3. 选择 **Pixel 6** → 下载 **API 34** 系统镜像
4. 启动模拟器

#### 4. 编译运行

1. 点击顶部工具栏的 **运行设备选择框**（显示 "No Devices" 或你的手机型号）
2. 选择已连接的手机或模拟器
3. 点击 **Run** 按钮（绿色三角形 ▶️）
4. 等待编译（首次约2-5分钟）
5. APP自动安装到手机并启动

#### 5. 获取APK文件

编译成功后，APK位于：

```
PhotoBorderApp/app/build/outputs/apk/debug/app-debug.apk
```

可以直接复制这个文件发给其他人安装。

## 方案二：GitHub Actions 自动编译

适合：没有Android Studio环境，或想自动编译

### 步骤

#### 1. 注册 GitHub 账号

1. 访问 https://github.com
2. 点击 **Sign up** 注册（邮箱验证）

#### 2. 创建新仓库

1. 登录 GitHub
2. 点击右上角 **+** → **New repository**
3. 填写信息：
   - Repository name: `PhotoBorderApp`
   - Description: `摄影边框APP`
   - 选择 **Public**（免费）
   - 勾选 **Add a README file**
4. 点击 **Create repository**

#### 3. 上传代码

**方式A：网页上传（简单，适合小项目）**

1. 进入刚创建的仓库页面
2. 点击 **Add file** → **Upload files**
3. 把 `PhotoBorderApp` 文件夹内的所有文件拖入（注意：不是拖文件夹，是拖文件夹内的文件）
4. 点击 **Commit changes**

> 注意：`.github/workflows/build.yml` 文件必须上传到正确位置

**方式B：Git命令行（推荐）**

```bash
# 1. 进入项目目录
cd PhotoBorderApp

# 2. 初始化Git仓库
git init

# 3. 添加所有文件
git add .

# 4. 提交
git commit -m "Initial commit"

# 5. 关联远程仓库（把下面的URL换成你的仓库地址）
git remote add origin https://github.com/你的用户名/PhotoBorderApp.git

# 6. 推送代码
git push -u origin main
```

#### 4. 触发自动编译

1. 代码推送后，点击仓库顶部的 **Actions** 标签
2. 看到 **Build APK** 工作流正在运行（黄色圆圈）
3. 等待约5-10分钟
4. 运行完成后变成绿色勾号 ✅

#### 5. 下载APK

1. 点击完成的 workflow 运行记录
2. 页面底部看到 **Artifacts** 区域
3. 下载：
   - `app-debug` → 调试版APK（推荐测试用）
   - `app-release` → 发布版APK（需签名才能安装）

#### 6. 安装到手机

1. 下载的APK文件传到手机（微信、QQ、数据线均可）
2. 手机上点击APK文件
3. 允许 **安装未知来源应用**
4. 完成安装

## 常见问题

### Q1: Gradle同步失败，提示网络错误

**解决：**
- 检查网络连接
- 尝试切换网络（手机热点）
- 在 `gradle.properties` 中添加国内镜像：
```properties
systemProp.http.proxyHost=mirrors.cloud.tencent.com
systemProp.http.proxyPort=80
```

### Q2: 编译报错 "Could not find method xxx()"

**解决：**
- 检查 Android Studio 版本是否最新
- 点击 **File → Invalidate Caches → Invalidate and Restart**

### Q3: 手机连接后Android Studio不识别

**解决：**
- 检查USB线是否支持数据传输（有些线只充电）
- 手机上重新允许USB调试
- 尝试更换USB接口

### Q4: GitHub Actions编译失败

**解决：**
- 检查 `.github/workflows/build.yml` 文件是否在正确位置
- 检查代码是否完整上传
- 点击失败的workflow查看详细日志

### Q5: 安装APK提示"应用未安装"

**解决：**
- 发布版APK需要签名，先用调试版测试
- 检查手机Android版本是否8.0+
- 卸载旧版本后重新安装
